DEPLOYMENT IN 5 STEPS
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Create GitHub Repository (5 minutes)
─────────────────────────────────────────────
  1. Go to https://github.com/new
  2. Name: "prompt-enhancer"
  3. Choose Public or Private
  4. Create repository
  5. Clone it locally
  6. Copy ALL files from d:\ai prompt optimiser\ to your repo
  7. git add . && git commit -m "Initial commit" && git push origin main

STEP 2: Sign Up for Render (2 minutes)
──────────────────────────────────────
  1. Go to https://render.com
  2. Click "Sign up"
  3. Select "Sign up with GitHub"
  4. Authorize Render to access your GitHub account

STEP 3: Connect Repository (1 minute)
──────────────────────────────────────
  1. Grant Render access to prompt-enhancer repository
  2. Render will detect your repository

STEP 4: Create Web Service (2 minutes)
───────────────────────────────────────
  1. In Render dashboard: New → Web Service
  2. Select "prompt-enhancer" repository
  3. Configure:
     Name: prompt-enhancer
     Environment: Python 3.11
     Build Command: pip install -r requirements.txt
     Start Command: python -m streamlit.cli run app.py --server.port=$PORT --server.address=0.0.0.0
     Plan: Free
  4. Click "Create Web Service"

STEP 5: Wait for Deployment (3-5 minutes)
──────────────────────────────────────────
  1. Render builds and deploys your app
  2. Watch the logs for progress
  3. Once complete: "Your service is live"
  4. Get your URL: https://prompt-enhancer.onrender.com

Total Time: ~15 minutes from GitHub to live! 🚀

═══════════════════════════════════════════════════════════════════════════════
TROUBLESHOOTING RENDER DEPLOYMENT
═══════════════════════════════════════════════════════════════════════════════

Issue: Build fails
Solution:
  • Check Render logs: Dashboard → Logs tab
  • Verify all packages in requirements.txt are spelled correctly
  • Ensure Python version is 3.11

Issue: App won't start
Solution:
  • Check logs for specific errors
  • Verify Procfile format exactly
  • Test locally first

Issue: "Service Unhealthy"
Solution:
  • First start takes 30-60 seconds
  • Service may be downloading models
  • Give it 2-3 minutes
  • Subsequent starts are faster

Issue: Very slow performance
Solution:
  • Free tier may be slower due to shared resources
  • Upgrade to paid tier if needed
  • Or accept that first load takes 30-60 seconds
